# AIDeckPP 記法リファレンスシート

このリファレンスシートは、AIDeckPPでサポートされている **Markdown**、**AsciiDoc**、および **YAML** 形式の記法をまとめたものです。それぞれの形式において「AIDeckPP特有の機能」と「共通のフォーマット機能」に分類して整理しています。

---

## 1. Markdown（マークダウン）記法

### AIDeckPP特有の機能（スライド制御・変数等）

*   **グローバル（全体）設定 (YAMLフロントマター)**:
    ファイルの先頭に `---` で囲まれたブロックを記述することで、スライド全体の共通設定を行うことができます。
    ```yaml
    ---
    footer: "全スライド共通のフッターテキスト"
    date: "日付（例: 2026-07-16）"
    toc: true # 自動目次スライドの生成（true/false）
    toc_title: "目次" # 目次スライドのタイトル名（任意）
    theme: "default" # 使用するテーマ名（任意）
    ---
    ```
*   **スライドの区切りとタイトル**:
    *   `# スライドタイトル`: スライドの開始と、そのスライドのタイトルを定義します。
    *   `## セクション` / `### 見出し`: スライド内のセクションや見出し（`####` 未満はスライド分割としても扱われます）。
*   **自動ナンバリング・プレースホルダー自動入力**:
    テンプレート（スライドマスターやレイアウト）内のテキストボックス（プレースホルダー）名に、以下の特定のキーワードを指定しておくと、スライド生成時に自動的に値が入力されます。
    比較時、**大文字小文字は区別されず、空白や `_`（アンダースコア）の有無も無視**されます（例: `slide_no` と `slideno` は同じと判定されます）。
    *   `slideno` (または `slide_no`): 1から始まる現在のスライド番号を自動入力。
    *   `sectionno` (または `section_no`): 新しいセクション（`##` から始まるスライド）が出現するたびに自動でインクリメントされる章番号を自動入力。
    *   `footer`: グローバル（全体）設定（YAMLフロントマター）の `footer` に指定した共通フッターテキストを自動入力。
    *   `date`: グローバル（全体）設定（YAMLフロントマター）の `date` に指定した共通日付テキストを自動入力。
    *   `title`: スライドのタイトルを自動入力。
    *   `subtitle`: スライドのサブタイトルを自動入力。
*   **PowerPoint標準の自動スライド番号フィールド**:
    *   テキスト内に `<#>` と直接記述すると、PowerPoint標準のスライド番号表示フィールドとして機能し、スライド表示時に自動的に現在のページ番号に変換されて描画されます。
*   **コメントパラメータによる制御**:
    HTMLのコメント構文を使ってスライド単位のパラメータを制御します。1行にセミコロン（`;`）で区切って複数並べることも可能です。
    **※重要: コメントパラメータは、必ず「対象となる行や要素の直前の行」に記述する必要があります。**（例: スライドレイアウトを適用したい場合は、`# スライドタイトル` の直前の行に記述します）
    *   `<!-- layout "レイアウト名" -->` (または `<!-- l "レイアウト名" -->`): スライドのレイアウト（マスター）を指定します。
    *   `<!-- subtitle "サブタイトル" -->` (または `<!-- sub "サブタイトル" -->`): サブタイトルを指定します。
    *   `<!-- placeholder "場所名" "値" -->` (または `<!-- ph "場所名" "値" -->`): テンプレート上の名前付きプレースホルダーに直接テキストを流し込みます。
    *   `<!-- align "配置値" -->` (または `content_align`, `valign`): スライド内コンテンツの垂直方向の配置位置を調整します。指定可能な値は以下の通りです。
        *   `top`: 通常より上に大きく移動（Y軸 -0.6インチ）
        *   `semi-top` (または `high`): 通常より上に少し移動（Y軸 -0.3インチ）
        *   `center`: 標準の基準位置（デフォルト）
        *   `semi-bottom` (または `low`): 通常より下に少し移動（Y軸 +0.3インチ）
        *   `bottom`: 通常より下に大きく移動（Y軸 +0.6インチ）
    *   `<!-- newpage "レイアウト名" -->` (または `new_page`, `new`, `n`): 明示的に改ページ（スライド分割）を行い、かつ新しく追加するスライドのレイアウト名を指定します（レイアウト名は省略可能です）。
    *   `<!-- gallery 列数 -->` (例: `<!-- gallery 3 -->`): 続く画像をギャラリー（グリッド）として自動配置します。
*   **マルチパネルレイアウト (Split)**:
    **※重要: splitコマンドも「対象となる要素（テキスト等）の直前の行」に記述する必要があります。**
    *   `<!-- split 方向 -->`: パネル分割を開始します。指定可能な方向は以下の通りです。
        *   `h` (または `horizontal`): 左右（水平）方向へのパネル分割（デフォルト）
        *   `v` (または `vertical`): 上下（垂直）方向へのパネル分割
    *   `<!-- panel -->`: 各パネル（カラム/行）のコンテンツ境界を定義します。
    *   `<!-- /split -->`: マルチパネル分割領域を終了します。
*   **ブロック要素**:
    *   **フローチャート (`flow` ブロック)**:
        ```markdown
        ```flow
        [開始] --> [処理]
        [処理] -> [終了]
        ```
        ```
    *   **比較マトリックス (`comparison` ブロック)**:
        ```markdown
        ```comparison title="表のタイトル（任意）"
        カラム1のラベル:
        - 項目A
        - 項目B
        カラム2のラベル:
        - 項目C
        - 項目D
        ```
        ```
    *   **タイムライン (`timeline` ブロック)**:
        ```markdown
        ```timeline
        日時1: イベント名 - 詳細説明
        日時2: イベント名 - 詳細説明
        ```
        ```
    *   **ツリー構造 (`tree` ブロック)**:
        ```markdown
        ```tree
        親ノード
          子ノード 1
            孫ノード
          子ノード 2
        ```
        ```
    *   **コードブロック (`code` ブロック)**:
        ```markdown
        ```code python
        print("Hello World")
        ```
        ```
        *※等幅フォント、薄グレー背景で描画され、上部に言語名が斜体でキャプション表示されます。キャリブレーション時はレベル0のフォントサイズが適用されます。*
    *   **プレーンコードブロック（言語指定なし）**:
        ```markdown
        ```
        Hello World
        ```
        ```
        *※上部の言語名キャプションを表示せず、背景枠とコードテキストのみを描画します。*
    *   **Mermaidダイアグラム (`mermaid` ブロック)**:
        ```markdown
        ```mermaid
        graph TD
          A --> B
        ```
        ```
        *※Mermaid CLIが必要。インストールされていない場合は自動的に CodeBlock（コードプレーンテキスト）としてフォールバック描画されます。*
    *   **引用 (Blockquote / Quote)**:
        ```markdown
        > 引用の1行目
        > 引用の2行目
        ```
        *※薄グレー背景枠の左端に、縦のグレー線（R153 G153 B153 / #999999）が描画されます。通常のフォント（Calibri等）を使用し、キャリブレーション時はレベル0のフォントサイズが適用されます。*

### 共通のフォーマット機能

*   **文字装飾**:
    *   `**太字**`
    *   `*斜体*`
    *   `***太字斜体***`
*   **箇条書きリスト**:
    *   `- 項目` / `* 項目` (スペースによるインデントで階層化が可能です)。
*   **テーブル (表)**:
    *   標準的なMarkdownパイプテーブル（例: `| ヘッダー | ... |`）。
*   **画像**:
    *   `![キャプション](画像パス)` (altテキストがキャプションとして保持されます)。

## 2. AsciiDoc（アスキーダック）記法

### AIDeckPP特有の機能（スライド制御・変数等）

*   **グローバル（全体）設定 (ヘッダー属性)**:
    ファイルの先頭にコロン（`:`）で囲まれた属性定義を行うことで、スライド全体の共通設定を行うことができます。
    ```asciidoc
    :footer: 全スライド共通のフッターテキスト
    :date: 日付（例: 2026-07-16）
    :toc: # 自動目次スライドの生成を有効にする場合、単に記述
    :toc_title: 目次
    :theme: default
    ```
*   **スライドの区切りとタイトル**:
    *   `= スライドタイトル`: スライドの開始と、そのスライドのタイトルを定義します。
    *   `## セクション` / `### 見出し`: スライド内のセクション見出し・スライド分割。
*   **自動ナンバリング・プレースホルダー自動入力**:
    Markdownと同様に、テンプレート内の以下の名称を持つプレースホルダー（大文字小文字・アンダースコア有無は不問）に自動的に値が流し込まれます。
    *   `slideno` (または `slide_no`): 1から始まる現在のスライド番号を自動入力。
    *   `sectionno` (または `section_no`): 新しいセクション（`==` から始まるスライド）が出現するたびにカウントされる章番号を自動入力。
    *   `footer`: グローバル（全体）設定（ヘッダー属性）のフッター文字列を自動入力。
    *   `date`: グローバル（全体）設定（ヘッダー属性）の日付文字列を自動入力。
    *   `title`: スライドタイトルを自動入力。
    *   `subtitle`: スライドサブタイトルを自動入力。
*   **PowerPoint標準の自動スライド番号フィールド**:
    *   テキスト内に `<#>` と直接記述すると、表示時に自動的に現在のページ番号に変換されて描画されます。
*   **コメントパラメータによる制御**:
    AsciiDocの単一行コメント記法を使って制御します。
    **※重要: コメントパラメータは、必ず「対象となる行や要素の直前の行」に記述する必要があります。**（例: スライドレイアウト指定なら `== スライドタイトル` の直前の行に記述します）
    *   `// layout "レイアウト名"` (または `// l "レイアウト名"`)
    *   `// subtitle "サブタイトル"` (または `// sub "サブタイトル"`)
    *   `// placeholder "場所名" "値"` (または `// ph "場所名" "値"`)
    *   `// align "配置値"` (または `content_align`, `valign`): スライド内コンテンツの垂直方向の配置位置を調整します。指定可能な値は以下の通りです。
        *   `top`: 通常より上に大きく移動
        *   `semi-top` (または `high`): 通常より上に少し移動
        *   `center`: 標準の基準位置（デフォルト）
        *   `semi-bottom` (または `low`): 通常より下に少し移動
        *   `bottom`: 通常より下に大きく移動
    *   `// newpage "レイアウト名"` (または `new_page`, `new`, `n`): 明示的に改ページ（スライド分割）を行い、かつ新しく追加するスライドのレイアウト名を指定します（レイアウト名は省略可能です）。
    *   `// gallery 列数`
*   **マルチパネルレイアウト (Split)**:
    **※重要: splitやpanelコメントも「対象となる要素の直前の行」に記述する必要があります。**
    *   `// split 方向`: パネル分割を開始します。指定可能な方向は以下の通りです。
        *   `h` (または `horizontal`): 左右（水平）方向へのパネル分割（デフォルト）
        *   `v` (または `vertical`): 上下（垂直）方向へのパネル分割
    *   `// panel`
    *   `// /split`
*   **ブロック要素**:
    *   **フローチャート (`[flow]` ＋ `----` フェンス)**:
        ```asciidoc
        [flow]
        ----
        [開始] --> [処理]
        ----
        ```
    *   **比較マトリックス (`[comparison]` ＋ `----` フェンス)**:
        ```asciidoc
        [comparison]
        ----
        カラム1:
        - 項目A
        ----
        ```
    *   **タイムライン (`[timeline]` ＋ `----` フェンス)**:
        ```asciidoc
        [timeline]
        ----
        日時: イベント - 説明
        ----
        ```
    *   **ツリー構造 (`[tree]` ＋ `----` フェンス)**:
        ```asciidoc
        [tree]
        ----
        親ノード
          子ノード
        ----
        ```
    *   **コードブロック (`[source, 言語名]` ＋ `----` フェンス)**:
        ```asciidoc
        [source,python]
        ----
        print("Hello")
        ----
        ```
    *   **プレーンコードブロック (`++++` パススルーフェンス)**:
        ```asciidoc
        ++++
        print("Hello")
        ++++
        ```
        *※キャプションを表示しないコード背景枠を描画します。*
    *   **引用 (`[quote]` ＋ `____` フェンス)**:
        ```asciidoc
        [quote]
        ____
        引用の1行目
        引用の2行目
        ____
        ```
        *※4つのアンダースコア（`____`）で囲みます。グレー背景枠の左端に縦のグレー線を描画します。*

### 共通のフォーマット機能

*   **文字装飾**:
    AsciiDocでは、フォーマット対象の周囲のスペースの有無によって、使用する演算子の数が異なります。
    *   **左右にスペースがある場合 (Constrained markup)**: 1つの演算子（`*` や `_`）で囲みます。
        *   `*太字の語句*`
        *   `_イタリックの語句_`
        *   `*_太字のイタリックの語句_*` (太字斜体)
    *   **スペースが無い場合 (Unconstrained markup / 単語の一部など)**: 2つの演算子（`**` や `__`）で囲みます。
        *   `**太**字の文**字**`
        *   `__イタ__リックの文__字__`
        *   `**__太字のイタ__**リックの文**__字__**` (太字斜体)
*   **箇条書きリスト**:
    *   `* 項目` / `- 項目`
*   **テーブル (表)**:
    *   標準的なAsciiDoc PSVテーブル (`|===` で囲む) および CSV形式 (`[format="csv"]`)。
*   **画像**:
    *   `image::画像パス[キャプション]`

## 3. YAML（ヤムル）記法

### AIDeckPP特有の機能（スライド制御・変数等）

*   **グローバル（全体）設定**:
    YAMLファイルのルート（トップ）レベルにキーを定義することで、スライド全体の共通設定を行います。
    ```yaml
    toc: true # 自動目次スライドの生成オン/オフ
    toc_title: "目次" # 目次タイトル名
    footer: "全スライド共通のフッターテキスト"
    date: "日付（例: 2026-07-16）"
    theme: "default" # 使用するテーマ名
    ```
*   **スライド単位の設定**:
    *   `slides`: スライドオブジェクトの配列。
    *   `title`: スライドのタイトル。
    *   `subtitle`: サブタイトル。
    *   `layout_hint`: 使用するスライドレイアウト名。
    *   `content_align`: スライド内コンテンツの垂直配置位置。
        *   指定可能値: `top`, `semi-top` (または `high`), `center`, `semi-bottom` (または `low`), `bottom`
*   **自動ナンバリング・プレースホルダー自動入力**:
    テンプレート内の特定のキーワード名を持つプレースホルダー（大文字小文字・アンダースコア有無は不問）に自動的に値が流し込まれます。
    *   `slideno` (または `slide_no`): 1から始まる現在のスライド番号を自動入力。
    *   `sectionno` (または `section_no`): YAML構造上の各スライドに割り振られたセクション番号（自動計算）を自動入力。
    *   `footer`: 全体設定で指定されたフッター文字列を自動入力。
    *   `date`: 全体設定で指定された日付文字列を自動入力。
    *   `title`: スライドタイトルを自動入力。
    *   `subtitle`: スライドサブタイトルを自動入力。
*   **PowerPoint標準の自動スライド番号フィールド**:
    *   テキスト内に `<#>` と直接記述すると、表示時に自動的に現在のページ番号に変換されて描画されます。
*   **要素定義 (`elements` 配列)**:
    *   **マルチパネルレイアウト (Split)**:
        ```yaml
        - type: Split
          direction: h # パネル分割方向。h (horizontal / 左右) または v (vertical / 上下)
          panels:
            - title: カラム1のタイトル
              elements:
                - type: Text
                  text: 本文テキスト
            - title: カラム2のタイトル
              elements: ...
        ```
    *   **フローチャート (Flow)**:
        ```yaml
        - type: Flow
          code: |
            [開始] --> [処理]
        ```
    *   **比較マトリックス (Comparison)**:
        ```yaml
        - type: Comparison
          title: "タイトル"
          columns:
            - name: カラム1
              items: ["項目A", "項目B"]
        ```
    *   **タイムライン (Timeline)**:
        ```yaml
        - type: Timeline
          events:
            - date: "日時"
              title: "タイトル"
              description: "詳細説明"
        ```
    *   **ツリー構造 (Tree)**:
        ```yaml
        - type: Tree
          root:
            label: 親
            children:
              - label: 子
        ```
    *   **コードブロック (CodeBlock)**:
        ```yaml
        - type: CodeBlock
          code: |
            print("Hello")
          language: python # 省略時はキャプション非表示
          caption: "任意の説明文"
        ```
    *   **Mermaidダイアグラム (Mermaid)**:
        ```yaml
        - type: Mermaid
          code: |
            graph TD
              A --> B
        ```
    *   **引用 (Quote)**:
        ```yaml
        - type: Quote
          text: |
            引用の1行目
            引用の2行目
        ```

### 共通のフォーマット機能

*   **本文テキスト (Text)**:
    ```yaml
    - type: Text
      text: "テキスト内容"
    ```
*   **箇条書きリスト (BulletList)**:
    ```yaml
    - type: BulletList
      bullets:
        - text: "項目内容"
          level: 0 # 0から始まるインデント階層
    ```
*   **画像 (Image)**:
    ```yaml
    - type: Image
      source: "画像パス"
      caption: "キャプション"
    ```
*   **テーブル (Table)**:
    ```yaml
    - type: Table
      headers: ["列1", "列2"]
      rows:
        - ["データA", "データB"]
    ```
